Progressive Web Apps


are regular web pages or websites, but can appear to the user like traditional applications or native mobile applications.
This application type attempts to combine features offered by most modern browsers with the benefits of a mobile experience.

Features

Progressive Web Apps are user experiences that have the reach of the web, and are:

Reliable

When launched from the user’s home screen, service workers enable a Progressive Web App to load instantly, regardless of the network state.

Engaging

They are installable and live on the user's home screen, without the need for an app store, offer an immersive full screen experience and can even re-engage users with web push notifications.

Fast 

53% of users will abandon a site if it takes longer than 3 seconds to load! And once loaded, users expect them to be fast—no janky scrolling or slow-to-respond interfaces.

Requirements for a Progressive Web App

Responsive

It needs to fit any form factor, from traditional computers to tablets and smartphones

Connectivity independent

Progressively-enhanced with Service Workers to let them work offline and synchronize later

App-like-interactions

Adopt a Shell + Content application model to create appy navigations & interactions

Fresh

Transparently always up-to-date thanks to the Service Worker update process

Safe

Served via TLS (a 
Worker requirement) to prevent snooping

Discoverable

Are identifiable as “applications” allowing search engines to find them

Re-engageable

Can access the re-engagement UIs of the OS

Installable

To the home screen through browser-provided prompts

Linkable

Meaning they’re zero-friction, zero-install, and easy to share
